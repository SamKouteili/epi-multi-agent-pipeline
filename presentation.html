The EPI Has a Data Problem

  • The EPI ranks 180 countries across 58+ indicators spanning environmental health and ecosystem vitality
  • Many indicators suffer from poor country coverage, infrequent updates, or reliance on modeled data
  • Example: WRR (Waste Recovery Rate) — 56 of 180 countries imputed via regression with only R²=0.42
  • Example: WWR (Wastewater Reused) — data exists for a single year (2015) for most countries
# What the EPI team often faces # ───────────────────────────── Indicator Coverage Imputed Issue ─────────────────────────────────── WRR 124/180 56 R²=0.42 SMW 124/180 56 same data WPC 124/180 56 same data WWR ~80/180 100+ 1 year only PRS 180/180 0 2 timepoints BER 180/180 0 5 timepoints # Goal: find alternative data # sources that correlate with # these hard-to-measure indicators

Data Proxies: The Core Idea

A data proxy is a measurable variable that correlates with a hard-to-measure target. If the correlation is strong and not spurious, the proxy can supplement or validate official statistics.

# Classic example from the proposal: # ─────────────────────────────────── anti_diarrheal_sales ~ water_quality_problems # Pharma sales data is cheaper, # more frequent, broader coverage # than direct water quality testing

The framework leverages modern AI to access expert insights alongside traditional reasoning engines and statistical analyses to uncover such connections at scale.

# Each hypothesis is formalized as: h = ψ(context, variables, relationship) context: { geographic_scope: "global", time_period: "2010-2021" } variables: { target: "UWD", # unsafe drinking water proxy: "handwashing_access" } relationship: { direction: "negative", form: "log-linear", strength: "r = -0.65" }

Verdict Decision Tree:

if direction mismatch → rejected
if p > 0.10 → rejected
if 0.05 < p ≤ 0.10 → inconclusive

Fix 3: Controlling for Confounders

Problem: spurious correlations from confounding variables
Many environmental indicators correlate with GDP. Without controlling for it, nearly anything that tracks national wealth will appear to predict waste recovery, water treatment, etc.
Solution: mandatory partial correlation against standard controls
Every verification now tests run_partial_correlation(df, x_col, y_col, covar_cols) -> Optional[PartialCorrelationResult]Uses pingouin.partial_corr(). Returns None if n < 30. Returns partial_r, partial_p, control_variables.() against these 4 controls:
# src/utils/data_fetch.py LOCAL_CONTROLS: dict[str, str] = { "GPC": "GDP per capita, PPP", "HDI": "Human Development Index", "POP": "Total population", "URB": "Urban population (%)", } # A proxy is only useful if it # explains variance BEYOND these. # # WRR diagnostic showed exactly this: # URB→WRR: r=0.489 (looks good!) # URB→WRR | GPC: partial_r = -0.016 # (entirely confounded by GDP)

GDP, HDI, urbanization, and population all strongly predict many environmental indicators. The partial correlation step is what separates confirmed from partially_confirmed.

Why This Design: Toward a Knowledge Graph

We are building a knowledge graph of data proxies where nodes are data variables and edges are typed, quantified relationships.

By constraining edges with direction, strength, and functional form (all fields of Relationshipclass Relationship(BaseModel)direction: Direction # positive|negative|nonlinear functional_form: FunctionalForm # linear|log-linear|quadratic|... strength_estimate: Optional[str] # e.g. "r=0.72"), we can use probabilistic model checking to self-validate the graph.

Sign Consistency Check:

# Transitive consistency: # if A --[+]--> B and B --[+]--> C # then A --[?]--> C should be positive # # If the graph says A --[-]--> C, # something is wrong: likely a # confounded or misspecified edge. Example: GPC --[+]--> WWT # richer = more treatment WWT --[+]--> H2O # treatment = cleaner water GPC --[?]--> H2O # should be positive # If GPC --[-]--> H2O found, # flag for investigation

Self-Validation via Constraint Checking

With edges typed by Relationshipclass Relationship(BaseModel)direction: Direction functional_form: FunctionalForm strength_estimate: Optional[str] fields, we can define consistency constraints and automatically flag violations:

# Constraint 1: Sign Transitivity for path in graph.all_paths(A, C): expected = product( [e.direction for e in path]) if expected != direct_edge(A,C): flag("sign inconsistency") # Constraint 2: Strength Bounds # if A->B strong (r=0.8) and # B->C strong (r=0.7), # A->C can't be near-zero # Constraint 3: Confounder Detection # partial_r drops to ~0 after # controlling a 3rd variable # => edge is confounded, not causal

Concrete Example:

# From our WRR diagnostic run: # ──────────────────────────── GPC --[+, r=0.757]--> WRR # confirmed (partial r=0.662) URB --[+, r=0.489]--> WRR # partially confirmed GPC --[+]--> URB # known strong positive # Model checker finds: # URB->WRR has partial_r = -0.016 # after controlling GPC # => edge is CONFOUNDED, not causal # => remove or downgrade URB->WRR

The structured hypothesis format (ProxyHypothesisclass ProxyHypothesis(BaseModel)id: str context: Context target_variable: str proxy_variable: str relationship: Relationship mechanism: str data_source: DataSource confidence: Confidence evidence_type: EvidenceType caveats: list[str]) was designed specifically to enable this kind of graph-level reasoning.
